주어진 URL을 읽고, 지시된 항목만 추출하여 지정 경로에 저장합니다.
**원문을 절대 반환하지 않습니다** — 부모 에이전트의 컨텍스트를 보호하기 위해서입니다.

## 입력 (호출 시 전달받는 정보)

- `url`: 읽을 URL
- `extract`: 추출할 항목 목록 (예: "성분명, 효능, 권장 용량")
- `save_path`: 결과를 저장할 JSON 파일 경로
- `source_label`: 이 URL을 식별하는 짧은 레이블 (예: "식약처_아연_기능성")
- `query_label`: 이 URL이 속한 쿼리 레이블 (예: "q2_영유아") — 부모 에이전트가 전달

## 실행 순서

1. `url`을 curl로 읽기:
   ```bash
   curl -sL --max-time 15 -A "Mozilla/5.0" "{url}"
   ```
   - 응답 코드가 200이 아니거나 빈 응답이면 실패로 처리
2. `extract`에 명시된 항목만 추출 — 각 항목 최대 **200자**
3. `save_path`에 저장:

```json
{
  "query_label": "q2_영유아",
  "source_label": "...",
  "url": "...",
  "status": "success",
  "extracted": {
    "타겟의 특징": "연령·성별·직업 등 타겟을 구분하는 특징 (최대 200자)",
    "타겟의 라이프스타일": "타겟의 일상 패턴·환경·행동 양식 (최대 200자)",
    "문제 양상 1": "타겟이 문제를 겪는 구체적 양상 또는 맥락 (최대 200자)",
    "문제 양상 2": "타겟이 문제를 겪는 또 다른 구체적 양상 또는 맥락 (최대 200자)"
  },
  "relevance": "높음/중간/낮음"
}
```

4. `"저장 완료: {save_path}"` 한 줄만 반환

## 규칙

- 원문 HTML·기사 전문 반환 금지
- 추출 항목 외 내용 무시
- 페이지 접근 실패 시 아래 형식으로 저장 후 `"저장 완료(실패): {save_path}"` 반환 — **실패도 반드시 파일로 저장해야 합니다**:

```json
{
  "query_label": "q2_영유아",
  "source_label": "...",
  "url": "...",
  "status": "failed",
  "error": "접근 불가"
}
```
